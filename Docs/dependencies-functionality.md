# Dependencies

This project relies on several dependencies to provide various functionalities. Below is a list of the main dependencies
used in this project along with a brief description of each:

## @faker-js/faker

@faker-js/faker is a library that provides a simple and powerful way to generate fake data. It can be used to generate
random names, addresses, phone numbers, and more. This library is commonly used for testing and development purposes.

## @playwright/test

@playwright/test is the official testing framework for Playwright, a powerful browser automation library.
It provides a simple and intuitive API for writing end-to-end tests and supports multiple browsers including Chromium, Firefox, and WebKit.

## @types/node

@types/node is a TypeScript type definition library that provides type definitions for Node.js.
It allows TypeScript to understand the types used in Node.js and provides type checking and autocompletion for Node.js code.

## @dotenv

@dotenv is a library that allows you to load environment variables from a .env file. It is commonly used for storing sensitive information
such as API keys and database credentials.

## @express

express is a popular web framework for Node.js. It provides a simple and flexible way to build web applications and APIs.
It supports routing, middleware, and template engines, making it easy to build robust and scalable web applications.

## @playwright-cli-select

@playwright-cli-select is a command-line interface for Playwright that allows you to run tests with different configurations.
It provides options to run tests in parallel, with different browsers, and with different test suites.

## @ts-node

@ts-node is a TypeScript execution environment that allows you to run TypeScript code directly without the need to compile it first.
It provides type checking and autocompletion for TypeScript code and can be used for development and testing purposes.

## @typescript

@typescript is a superset of JavaScript that adds static typing to the language. It provides type checking and autocompletion
for JavaScript code and can be used to build large-scale applications with confidence.

## @winston

@winston is a logging library for Node.js that provides a simple and flexible way to log messages to various destinations such as
files, databases, and more. It supports different log levels, formatting, and transports, making it easy to customize logging behavior
for different applications.
